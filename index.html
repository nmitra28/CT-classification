OCI Data Science Tutorial

with Use Cases khan993@umd.edu

OCI’s data science cloud platform is built for teams allowing them to rapidly build, train, deploy, and manage machine learning models.
In this tutorial, we will walk you through the procedure of building and deploying your Machine Learning model. We will cover the entire pipeline of using OCI and specifically cover the following steps:

1. Setting Up (VCN, Project, Jupyter Lab notebook Session)
2. Understanding and Using Conda Environment and Environment Explorer
3. Use-Cases:
a. Getting Data from Object Storage
b. EDA and others using ADS
c. ML model train, test, implementation using AutoML
d. Saving model to Model Catalog
e. Deploying ML model as a CLI, REST API on OCI

Setting Up

Prerequisite Setup

(Compartment and VCN)

1. Compartment
This tutorial has two use cases using multiple OCI services (Data Science Platform, ADS, AutoML etc). We assume that you are in a Region with all of the needed services required. This project was written in the US East (Ashburn) region. It also assumes that you've already defined a compartment (see how to do that here) and your compartment has access to the services.

2. Virtual Cloud Network (VCN)
In order to create any Data Science Project and Jupyter Notebook in OCI, one needs to create a Virtual Cloud Network where the notebook is available. For this tutorial, we used a VCN wizard to create the needed network. To make it easier to set up a VCN, OCI provides a wizard to do that for you. You can check how to create a VCN using the wizard here).

Start w/ Data Science Platform

(Create Project and Notebook session)

1. Create a Project
In the Data Science Platform, first you would need to create a Project. Below we have provided the steps to do that.

a. Log into the Oracle Cloud.
b. Use the top-left menu to select the "Analytics & AI > Data Science" option or search up “Data Science”
c. Select the compartment of interest and click the “Create Project” sign.
d. Fill in the name, details and/or tag the project, click Create. Now you have made your project.


2. Make your Notebook Session
Now that you have created your Project, you can create multiple notebook sessions.

a. Inside the project, you can make multiple notebook sessions with user-specified names, compute instances and block storage space. Here, you would need the VCN and subnet you wish to create your session in. If you used the VCN Wizard earlier, your private subnet has been created for you to use.


b. Wait for your notebook session to create. select the "{notebook session} > open" option.
Now you have successfully created a notebook session.

Conda Environments and Environment Explorer

Choose the right Environment

Environment Explorer and Adding environment using Terminal

The Data Science service offers a series of prebuilt conda environments, and you can access them in the JupyterLab Launcher tab by clicking Environment Explorer to open in a new tab. It allows you to:

  • Browse existing conda environments.
  • Search conda environments
  • Install prebuilt conda environments in your notebook session using the provided instructions.
  • When a conda is installed, you can clone it. You can also look at the multiple notebooks examples for each environment installed.
The environments are focused on providing specific tools and a framework (for example, PySpark) to do machine learning work (for example, General Machine Learning for GPUs). Or providing a comprehensive environment to solve business use cases. For example, for our Fraud Detection use case we used General Machine Learning for CPUs.
